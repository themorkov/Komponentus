# Komponentus

**Komponentus** — консольное приложение для перевода и сопоставления автомобильных лакокрасочных миксов между различными системами производителей.
Проект ориентирован на быстрый поиск аналогов компонентов из одной системы в другую через текстовый интерфейс.
Целевая аудитория - колористы и магазины автоэмалей.

---

## Реализованные возможности

- Перевод компонентов между 36 системами ЛКМ:
Spies Hecker, Standox, Brulex, Profiline, Mobihel, Normex, Nason, Loritone,Kiwix, Vika,
Duxone, General, Baslac, Spectral, Green Line, R‑M, DuPoint,PPG, Quick Line, Autolux,
Maxmaeyer, REIZ, Miramishi, Yatu EasiCoat, Hebake, Kapci, Sikkens, Lesonal, DYNA, 
Debeer, Kansai Paint, Cardea, Rock Paint, KCC, Maxmeyer, Glasurit 55 line.
- Интерактивный консольный интерфейс
- Проверка корректности пользовательского ввода

---

## Архитектура приложения и используемые технологии и требования

Проект реализован на языке программирования C++ с использованием архитектурного паттерна **MVC (Model–View–Controller)**:

- **Model**  
  Хранение таблиц соответствий миксов в двумерном массиве (`std::string_view`)

- **View**  Windows Console View
  Отображение информации пользователю и оформление консольного интерфейса  

- **Controller**  
  Обработка пользовательского ввода и управление логикой взаимодействия

Для улучшения производительности и экономии оперативной памяти был использован механизм просмотра данных без копирования std::string_view.

Приложение работает только в Windows (из-за использования std::string_view версия языка должна быть не ниже C++17.
Данные миксов заданы статически нет возможности изменения и добавления
Нет поиска с частичным совпадением
